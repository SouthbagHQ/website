Southbag Institutional Services Ltd. announces **Southbag Code**, a terminal-native coding agent for developers who prefer to remain where they already work. The product is available globally via npm. Kevin has observed the launch.

Southbag Code is built around a small surface area and a serious workflow. The interface stays close to the command line. The agent handles the rest. It does not need to make a spectacle of itself.

## Installation

Southbag Code installs globally and runs from the project working directory:

```bash
npm i -g @southbag/code
southbag-code
```

No additional onboarding ceremony is required. Open a repository, start the agent, and describe the task in plain language. It will begin.

## Project aware

Southbag Code loads local instruction files, settings, themes, skills, and extensions from the project. Context travels with the repository rather than with a separate application shell. The repo knows what it wants. Southbag Code listens.

## Session native

Work is organised around sessions that can be continued, resumed, forked, cloned, compacted, imported, exported, and inspected. Developers may return to recent sessions, branch from earlier points in a conversation, or export a session when work needs to travel between environments. Nothing important has to disappear just because the terminal was closed.

## The workflow

The intended flow is direct:

1. **Start** — Run Southbag Code in the project directory and state the objective.
2. **Steer** — Queue follow-ups while the agent works, reference files with `@`, and run shell commands inline.
3. **Return** — Review the diff, resume when needed, and keep context intact across sessions.
